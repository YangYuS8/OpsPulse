# PROJECT_NOTE

做这个项目的原因很简单。

我想要一个能放进 homelab 里、自己看着顺手的心跳控制台。不是大而全的监控平台，也不是 Prometheus 的替代品，只是一个能让我快速知道机器是不是活着、资源是不是正常、服务和容器有没有明显异常的小工具。

当前做到这里，已经能用了：

- 宿主机 Agent 主动上报心跳
- Server 用 SQLite 存最新状态和一小段历史
- 前端用终端风格展示节点、资源、服务、容器和事件
- 不需要在节点上额外暴露端口

v0.1 先停在这里，是因为再往后做，很容易滑向另一个方向：

- 告警系统
- 权限系统
- 远程执行
- 更复杂的采集协议
- 更大的监控平台

这些都不是这个版本要解决的问题。

这个版本更像一个能展示、能运行、也能封存的基线：
它说明 OpsPulse 这条路是可行的，但还没有膨胀成一个重系统。

后面如果还继续扩展，比较自然的方向可能是：

- 更好的历史趋势展示
- 更细的容器和检查筛选
- 更明确的离线/退化事件归类
- 更适合长期运行的配置样例
- 更完整的宿主机部署说明

但这些都可以留给下一个版本，不必塞进 v0.1。
